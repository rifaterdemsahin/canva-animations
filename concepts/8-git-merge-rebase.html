Concept: Git Merge vs. Rebase

üß† The Concept

`git merge` and `git rebase` are two ways to integrate changes from one branch into another. `merge` creates a new "merge commit" that ties the two histories together. `rebase` rewrites history by replaying commits from the feature branch on top of the main branch, resulting in a linear history.

  3. Rebase: Animate the feature branch's commits lifting off and re-attaching one-by-one to the end of the main branch using "Pop" or "Drift".
